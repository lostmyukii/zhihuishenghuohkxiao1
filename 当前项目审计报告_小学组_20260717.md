## 三、当前最强的比赛证据

### 1. 真实数据链

```text
光敏/DHT11/声音/PIR
  -> N16R8本地采集
  -> 115200单行JSON telemetry
  -> OLED与Dashboard同步
```

Dashboard只有收到新鲜`hello/telemetry`才显示开发板在线；遥测停止约3.5秒后恢复离线和“等待实时数据”，不会拿默认值冒充真板。

### 2. 学习灯自动控制链

```text
学习模式 + 光照低于阈值
  -> 固件本地判断
  -> D12/GPIO12继电器与低压学习灯动作
  -> Dashboard和OLED同步
```

该链已经由遮挡/放开GPIO1光敏传感器真板验收，属于“智能控制”最稳定的主展示。

### 3. 噪声与温度提醒链

- 学习模式拍手超过声音阈值：网页橙色提醒、蜂鸣器响、OLED显示`BZ:ON`；停止后自动解除。
- 温度`27.1 C`时把阈值临时改为`26 C`：收到`ack set=temperatureThreshold`，网页显示当前值和阈值，蜂鸣器与OLED同步；恢复`29 C`后提醒解除。

### 4. 节能响应链

```text
暗处 + 无人 -> 关灯
暗处 + 有人 -> 开灯
再次无人 -> 自动关灯
```

页面同时解释“为什么开灯/关灯”。网页中的“节能分”是规则型评价，不是电表测得的瓦时或千瓦时；答辩应强调真实开关动作和节能原因，不能宣称精确测得耗电量。

### 5. 公网展示增强

- 公网地址：<https://hongkongxiao.ilelezhan.cn/>。
- 本次重新检查返回HTTPS 200。
- `index.html`、`app.js`、`style.css`、`intent-core.js`、`energy-core.js`、`cloud-core.js`、`alert-core.js`七个文件与本地SHA-256一致。
- 历史实板验收已证明：插板Mac的Chrome通过Web Serial把真板数据送到WSS/MQTT，无USB浏览器可实时观看并把模式命令回写到真板。

公网、WSS和MQTT是展示增强，不属于小学四项任务的替代品。顶部显示“WSS在线”不能单独证明开发板在线，必须同时看到新鲜真板帧。

## 四、当前真实性边界

### 可以直接说

- 四类核心传感数据来自N16R8真板，并同步到OLED和网页。
- D12/GPIO12低压学习灯已经完成网页手动控制和光照阈值自动控制。
- GPIO4噪声提醒、GPIO5离家提醒、GPIO14温度提醒都完成过实板触发与恢复。
- energy模式能依据光照和人体状态开关学习灯并说明原因。
- 远端浏览器与插板浏览器之间的WSS/MQTT双向链路完成过验收。
- 继电器只控制低压学习灯，不接220V。

### 只有再次现场预演通过后才能说

- 当前比赛电脑已经连接真实N16R8并稳定显示连续遥测。
- 所有传感器和D12学习灯在本次模型上仍能稳定复现。
- Windows电脑能够直接授权当前CH340 COM口。
- 另一台手机或电脑正在同步当前真板状态，而不是旧缓存或测试帧。

### 当前不能说

- “我说一句话，系统已经识别并执行。”当前只有文本意图模式。
- “语音交互任务已经完成。”在麦克风/STT真测通过前不能这样说。
- “节能分就是实际耗电量”或“节省了多少度电”。
- “声音百分比是标准分贝”。
- “当前作品有8键、RGB、风扇或舵机”。
- “旧连线图中的GPIO48仍是学习灯端口”。当前合同和真板证据都是`D12/GPIO12`。
- “网页能打开或WSS在线，就代表开发板在线”。
