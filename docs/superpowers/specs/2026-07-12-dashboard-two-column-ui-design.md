# Dashboard 浅蓝双栏操作台改版设计

日期：2026-07-12
参考页面：`https://zhinengshenghuo.ilelezhan.cn/`

## 1. 目标

把当前三列 Dashboard 改成适合小学组现场演示的浅蓝双栏操作台。改版借鉴参考页面的 iOS 状态色、半透明白色工作面板和“沙盘 + 学生操作台”结构，但不复制初中项目的硬件、导航、主题切换或大圆角装饰。

页面的唯一工作是让评委快速看见：真实开发板是否在线、传感器正在发生什么、当前模式如何判断、D12 学习灯为何动作，以及节能结果是什么。

## 2. 信息结构

桌面使用左宽右窄双栏，事件记录在底部跨满全宽：

```text
┌──────────────────────────────────────────────────────────┐
│ 项目名称                     USB / 开发板 / 帧 / 当前模式 │
├────────────────────────────────┬─────────────────────────┤
│ 学习小屋总览                   │ 学生操作台              │
│ 房间状态与真实传感器摘要       │ USB 连接                │
│                                │ 模式控制                │
├────────────────────────────────┤ 阈值调整                │
│ 传感器实时数据                 │ 学习灯控制              │
│ 光照 / 温湿度 / 噪声 / 人体    │ 节能分与中文原因        │
├────────────────────────────────┤ 语音意图                │
│ OLED 本地屏                    │                         │
├────────────────────────────────┴─────────────────────────┤
│ 事件记录                                                 │
└──────────────────────────────────────────────────────────┘
```

左栏顺序：

1. 学习小屋总览。
2. 传感器实时数据。
3. OLED 本地屏。

右栏使用“学生操作台”分组标题，顺序为：

1. USB 串口连接。
2. 模式控制与阈值调整。
3. 执行器、节能分和中文节能原因。
4. 语音意图。

底部为事件记录。保留现有全部功能和元素 ID，允许在 `index.html` 中增加左右栏包装元素并调整面板顺序，不重写 Web Serial 或业务逻辑。

## 3. 移动端顺序

在 `760px` 以下变为单栏，按现场操作优先级排列：

1. 顶部项目和实时状态。
2. USB 串口连接。
3. 模式控制与阈值。
4. 传感器实时数据。
5. 执行器和节能状态。
6. 学习小屋总览。
7. 语音意图。
8. OLED 本地屏。
9. 事件记录。

手机宽度 `390px` 时必须满足 `document.documentElement.scrollWidth === window.innerWidth`。按钮和中文原因必须换行，不允许遮挡相邻内容。

## 4. 视觉令牌

采用参考页面的冷静、清晰状态语言，同时避免单一蓝色主题：

| 角色 | 颜色 | 用途 |
| --- | --- | --- |
| 页面背景 | `#edf5fb` | 无渐变的浅蓝灰工作台底色。 |
| 主要文字 | `#15223a` | 标题和实时数值。 |
| 次要文字 | `#66748a` | 标签、GPIO 和说明。 |
| 系统蓝 | `#0a84ff` | 主要操作、当前模式和焦点。 |
| 在线绿 | `#30b866` | 在线、正常、节能成功。 |
| 等待橙 | `#e58a00` | USB 未连接和等待状态。 |
| 故障红 | `#e5484d` | 离线、失败和真实告警。 |

面板使用半透明白色 `rgba(255, 255, 255, 0.82)`、细边框、轻阴影和有限的 `backdrop-filter`。所有卡片和按钮圆角不超过 `8px`。不使用径向渐变、装饰光球或大面积紫蓝渐变。

字体继续使用系统中文字体栈。标题、正文和数据通过字号与字重区分，不随视口宽度缩放字号，字间距固定为 `0`。

## 5. 组件规则

- 顶栏：品牌信息在左，串口、开发板、帧数和模式状态在右；手机端自然换行。
- 面板标题：英文 eyebrow 降低存在感，中文标题是主要扫描点。
- 状态标签：保持短文本，绿色只表示新鲜在线数据，红色只表示真实故障。
- 模式按钮：四个按钮尺寸稳定，当前模式由真实 `telemetry.mode` 决定，不在点击瞬间伪装选中。
- 传感器：保留真实数值和 GPIO；布局紧凑但不嵌套额外卡片。
- 节能状态：与学习灯相邻，持续使用 `telemetry.energy`，离线显示“等待实时数据”。
- 学习小屋：保留三块房间状态，不加入初中户型图片或不存在的硬件热点。
- 事件记录：跨全宽，作为串口与命令证据，不占用第一操作路径。

## 6. 数据与错误状态

本改版不改变数据来源：

```text
N16R8 -> 115200 单行 JSON -> Web Serial -> 现有 app.js 状态 -> Dashboard
```

- 没有新鲜 `hello/telemetry` 时仍显示离线或等待。
- 不用 CSS 或默认 HTML 值制造在线、节能或传感器结论。
- `energy.score` 和 `energy.reason` 继续由已实现的 `energy-core.js` 处理。
- 页面改序后，所有 `querySelector` 对应 ID 必须存在且唯一。

## 7. 文件范围

| 文件 | 计划改动 |
| --- | --- |
| `dashboard/index.html` | 增加双栏包装、学生操作台标题并调整面板顺序；保留现有 ID。 |
| `dashboard/style.css` | 重建浅蓝双栏布局、状态色、面板样式和手机断点。 |
| `dashboard/app.js` | 仅在需要真实当前模式视觉状态时做小范围渲染增强。 |
| `dashboard/tests/` | 增加或更新 DOM/契约检查，确保关键 ID、脚本版本和节能区域未丢失。 |

如果修改 `app.js` 或新增脚本，必须更新 `index.html` 查询版本。当前没有 `dashboard/sw.js`，不新增 Service Worker。


浏览器验收：

1. `1440 x 1000`：左栏明显宽于右栏，学生操作路径从上到下完整可见。
2. `390 x 844`：单栏无横向滚动，按钮和中文节能原因不溢出。
3. 离线首屏：USB、开发板、节能状态均保持等待或离线，不出现模拟数据。
4. 实板连接：`hello/telemetry/ack`、模式按钮、D12 手动控制和节能原因功能不回归。
5. 控制台无 JavaScript 错误，键盘焦点清晰，`prefers-reduced-motion` 下无必要动画。

## 9. 范围边界

- 不增加顶部导航和主题切换器。
- 不复制参考站的沙盘图片或品牌图标。
- 不增加风扇、RGB、舵机、八键 AD 或其他不存在的硬件。
- 不修改固件 GPIO、自动化逻辑、串口协议、MQTT topic 或语音白名单。
- 不把 mock 数据当作实板验收结果。
