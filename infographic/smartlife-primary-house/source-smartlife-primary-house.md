## 1. 规则要求审计

小学组智慧生活不应做成初中组的功能堆叠版。规则材料更强调“生活场景合理、采集稳定、响应正确、交互清楚”。对应任务如下。

| 小学组任务 | 规则重点 | 本方案落点 |
| --- | --- | --- |
| 数据采集 | 采得到、读得稳、表达清楚。 | 采集光照、温湿度、噪声、人体状态，并在 OLED 与网页看板同步显示。 |
| 智能控制 | 根据预设光照或温度阈值自动触发生活设备。 | 光线暗自动开学习灯，温度高自动开风扇，噪声高短提醒。 |
| 语音交互 | 指令能精准触发功能，并能回读状态。 | 网页语音/小智语音发布安全白名单指令，如“开始学习”“开启节能”“现在光照多少”。 |
| 节能响应 | 在合适时机关掉或切换到节能状态，解释为什么省电。 | 无人或光照充足时关闭灯和风扇，网页显示节能分与节能原因。 |

规则侧风险：

- 不能只展示串口数字，必须让评委看见“传感器变化 -> 判断 -> 执行器动作 -> 页面同步”的闭环。
- 不能为了炫技把小学作品做成初中/高中复杂系统，核心任务仍然是四条小学能力链。
- 语音交互要稳定，命令数量宁可少，也要保证每条命令能触发真实状态变化。
- MQTT 和可视化属于展示增强项，必须服务于小学任务，不应喧宾夺主。

## 2. N16R8 套装要求审计

### 2.1 主控与扩展板

当前小学目录中真正适合本项目的是 `N16R8 ESP32-S3 + S3 传感器扩展板`，不是早期 UNO/Keyes 42 合一方案。UNO 资料可以借用场景叙事，但最终硬件合同应以 N16R8 为准。

N16R8 扩展板使用白色 `G-V-S` 插槽：

- `G`：GND，黑线。
- `V`：低压电源，红线。
- `S`：GPIO 信号，黄/白线。

所有模块必须共地。ESP32-S3 的 ADC/IO 信号不能超过 `3.3V`。

### 2.2 推荐硬件合同

小学组核心闭环建议只使用“看得见、讲得清、风险低”的模块。厨卫安全模块可以作为扩展，不作为第一版核心验收压力。

| 类型 | 模块 | 接口/GPIO | 方案角色 | 优先级 |
| --- | --- | --- | --- | --- |
| 感知 | 光敏传感器 | `ADC1 / GPIO1` | 数据采集、自动开灯、节能判断 | 必选 |
| 感知 | DHT11 温湿度 | `D14 / GPIO14` | 温度阈值、舒适判断、风扇控制 | 必选 |
| 感知 | 声音传感器 | `ADC4 / GPIO4` | 学习环境噪声提醒 | 建议 |
| 感知 | PIR 人体红外 | `D5 / GPIO5` | 有人/无人、节能响应、离家演示 | 必选 |
| 输入 | 8 键 AD 键盘 | `ADC3 / GPIO3` | 本地切换模式、调整阈值 | 建议 |
| 执行 | 继电器/学习灯 | `GPIO48` | 学习灯开关或低压灯带 | 必选 |
| 执行 | 风扇 PWM | `D11 / GPIO11`，可配 `D12 / GPIO12` 方向 | 温度高通风、节能风速 | 必选 |
| 执行 | 舵机窗帘 | `D9 / GPIO9` | 休息/节能场景下窗帘开合 | 建议 |
| 执行 | RGB 灯环 | `GPIO47` | 模式状态灯：学习蓝、节能绿、提醒黄、报警红 | 建议 |
| 执行 | 无源蜂鸣器 | `D13 / GPIO13` | 噪声/异常短提醒 | 建议 |
| 显示 | OLED | `SDA=GPIO41, SCL=GPIO42` | 本地数据显示与模式反馈 | 必选 |
| 安全扩展 | MQ-2 烟雾/燃气 | `ADC2 / GPIO2` | 厨房安全扩展 | 可选 |
| 安全扩展 | 水滴传感器 | `D8 / GPIO8` | 漏水提醒扩展 | 可选 |
| 安全扩展 | 火焰传感器 | `D6 / GPIO6` | 火源提醒扩展，不用明火演示 | 可选 |
| 身份扩展 | RFID RC522 | `SS=GPIO34, RST=GPIO35` | 门禁/家庭成员扩展 | 暂缓 |

重要差异：

- 初中项目中曾出现 `GPIO46 / GPIO12 / GPIO45` 等映射；小学当前本地 N16R8 文档更稳定地指向 `RGB=GPIO47`、`继电器=GPIO48`、`火焰=D6/GPIO6`。
- 本方案按小学目录中的 N16R8 文档写入。若更换实板批次，必须先用单模块例程复核丝印和端口。

### 2.3 安全约束

- 继电器只接低压演示负载，不接 `220V`。
- MQ-2 如用 `5V` 供电，`AO` 不能直接进入 ESP32-S3 ADC，必须分压/限压到 `0~3.3V`。
- 风扇或外部电源必须与 N16R8 共地。
- 火焰传感器不用明火演示，可用安全光源或按钮模拟。
- 水滴传感器只滴少量水，底部垫透明膜，避免泡坏模型。
- 上电默认：风扇关、继电器关、蜂鸣器静音、RGB 安全色。

## 3. 推荐作品定位

作品名称：**N16R8 智慧低碳学习小屋**

一句话定位：

> 这是一个面向小学生学习与生活的智能小屋。N16R8 会看光线、感知温度、听环境噪声、判断有没有人在房间里，并通过学习灯、风扇、窗帘、RGB 灯和 OLED 做出反馈；同时把真实数据通过 MQTT 和网页看板可视化，让评委看到每一次传感、判断和执行。

作品主线：

- 不是“智能家居大全”，而是“学生学习空间的舒适与节能助手”。
- 场景可落在一间学习房加客厅中控：书桌、窗户、风扇、灯、门口 PIR、中控 OLED。
- 四条小学任务全部能在 3 分钟内演示完。

## 4. 技术路线取舍

### 方案 A：推荐路线，Web Serial + WSS/MQTT + 可视化

```text
N16R8 USB
  -> HTTPS Dashboard Web Serial
  -> WSS Cloud Relay
  -> MQTT Broker
  -> 其他电脑/手机网页看板
```

优点：

- 沿用初中项目最稳的 MQTT 与可视化链路。
- 换电脑时只需要插 USB、用 Chrome/Edge 打开 HTTPS 网页、点击连接串口。
- 不需要把 Wi-Fi 名称和密码写死到固件里。
- 插板电脑能直接控制真实硬件，其他设备能远程看状态和下发安全命令。

限制：

- 插着 N16R8 的电脑浏览器页面必须保持打开。
- Safari、Firefox 通常不能做 USB 直连，只能观看远程状态。

### 方案 B：N16R8 Wi-Fi 直连 MQTT

```text
N16R8 Wi-Fi
  -> MQTT Broker
  -> Dashboard
```

优点：

- 看起来更无线化，硬件不依赖插板电脑。
- 和小学目录中 `N16R8_SmartHome` 的思路一致。

限制：

- 比赛现场 Wi-Fi、热点、密码、2.4GHz、认证页都会影响稳定性。
- 不建议写死家庭 Wi-Fi 和公共 broker。
- 需要增加配网机制或 USB 下发 Wi-Fi 配置，开发量更高。

### 方案 C：纯本地 OLED + 串口看板

```text
N16R8 USB
  -> 本地串口/本地网页
```

优点：

- 实现最快，适合调试。
- 断网也能演示核心传感与执行。

限制：

- 不能完整满足用户要求的 MQTT 与远程可视化。
- 平台感弱，不如初中功能迁移完整。

最终推荐：先做方案 A，保留方案 B 作为第二阶段。方案 C 只作为断网降级。

## 5. 功能设计

### 5.1 四个场景模式

| 模式 | 触发方式 | 自动逻辑 | 执行反馈 |
| --- | --- | --- | --- |
| 学习模式 `study` | 8 键 AD、网页按钮、语音“开始学习” | 光暗开灯；温度高开风扇；噪声高短提醒 | 学习灯、风扇、OLED、网页提示 |
| 休息模式 `rest` | 网页按钮、语音“休息模式” | 关闭学习灯，窗帘合上，噪声提醒静默 | 舵机窗帘、RGB 暖色 |
| 离家模式 `away` | 8 键 AD、网页按钮、语音“我要出门” | 关闭灯和风扇；PIR 有人则提醒 | RGB 红、蜂鸣器短报警、网页告警 |
| 节能模式 `energy` | 8 键 AD、网页按钮、语音“开启节能” | 无人或光照足时关闭灯；温度正常关闭风扇 | 节能分、节能原因、RGB 绿 |

小学核心演示建议聚焦 `study` 与 `energy` 两条，`rest` 和 `away` 作为补充。

### 5.2 小学任务映射

| 任务 | 演示动作 | 评委看到 |
| --- | --- | --- |
| 数据采集 | 遮挡光敏、热水杯靠近 DHT、拍手、PIR 前挥手 | OLED 和网页数值同步变化 |
| 智能控制 | 在学习模式下遮光或升温 | 灯自动亮，风扇自动转，页面显示触发原因 |
| 语音交互 | 说“开始学习”“当前光照多少”“开启节能” | 模式切换或语音记录出现，OLED/网页回显状态 |
| 节能响应 | 开启节能模式后离开 PIR 范围，或让光照充足 | 灯/风扇关闭，页面显示节能分提高 |

### 5.3 8 键 AD 设计

如果使用 8 键 AD，建议保持小学可讲清的映射：

| 键 | 功能 |
| --- | --- |
| A | 学习模式 |
| B | 休息模式 |
| C | 离家模式 |
| D | 节能模式 |
| 左/右 | 切换当前阈值：光照、温度、噪声、无人等待时间 |
| 上/下 | 当前阈值 `+5/-5` 或对应小步进 |

OLED 与网页必须显示“当前正在调哪个阈值”，避免评委看不懂按键在做什么。

## 6. MQTT 与可视化设计

### 6.1 项目 Topic

小学组建议使用独立 topic，避免和初中组互相污染：

```text
smartlife/primary/n16r8/hello
smartlife/primary/n16r8/telemetry
smartlife/primary/n16r8/event
smartlife/primary/n16r8/health
smartlife/primary/n16r8/command
smartlife/primary/n16r8/config
smartlife/primary/n16r8/voiceIntent
```

如果复用初中服务器代码，可在 relay 或 gateway 中做 topic 前缀配置，默认从 `smartlife/junior/n16r8` 改为 `smartlife/primary/n16r8`。

### 6.2 串口 JSON 协议

固件仍使用 `115200` 串口，每行一个 JSON，不混入调试文字。

启动 `hello` 示例：

```json
{"type":"hello","project":"smartlife-primary","board":"n16r8_esp32s3","profileId":"smartlife-primary-study-home-v1","firmware":"0.1.0","deviceName":"N16R8 智慧低碳学习小屋","baud":115200,"capabilities":["webSerial","mqttBridge","dashboard","voiceIntent","energyScore"],"pins":{"light":1,"sound":4,"dht":14,"pir":5,"keypad":3,"lamp":48,"fanPwm":11,"fanDir":12,"servo":9,"rgb":47,"buzzer":13,"oledSda":41,"oledScl":42}}
```

遥测 `telemetry` 示例：

```json
{"type":"telemetry","ts":123456,"mode":"study","sensors":{"light":31,"sound":22,"temperature":28.2,"humidity":55,"pir":true},"actuators":{"lamp":true,"fan":60,"curtain":45,"rgb":"blue","buzzer":false},"alerts":[],"energy":{"score":82,"reason":"光线不足，开启学习灯；有人在房间，保持舒适通风"},"health":{"profileId":"smartlife-primary-study-home-v1","mqtt":"bridge","voice":"ready","thresholdFocus":"lightThreshold"}}
```

命令示例：

```json
{"type":"command","mode":"study"}
{"type":"command","mode":"energy"}
{"type":"command","set":{"lightThreshold":35}}
{"type":"command","set":{"temperatureThreshold":29}}
{"type":"command","actuator":{"lamp":false}}
{"type":"voiceIntent","intent":"queryLight"}
{"type":"ping"}
```

固件收到命令必须返回 `ack`：

```json
{"type":"ack","ok":true,"message":"mode=energy"}
```

### 6.3 网页可视化首屏

首屏不要做介绍页，直接做操作台：

- 顶部状态：开发板、USB、WSS、MQTT、语音。
- 当前模式：学习、休息、离家、节能。
- 四个核心传感器：光照、温度/湿度、噪声、人体。
- 执行器：学习灯、风扇、窗帘、RGB、蜂鸣器。
- 小屋平面图热区：书桌、窗户、门口、中控台。
- MQTT/事件日志：只显示真实 `hello/telemetry/ack`，离线时明确提示“未收到真实数据”。
- 阈值面板：显示当前阈值、8 键 AD 焦点、网页滑块。
- 语音中心：语音输入、文本测试、最近一次识别、执行结果。

### 6.4 语音交互

第一阶段用网页语音或小智语音节点转成安全白名单，不让自由文本直接控制硬件。

| 语音 | 意图 | 动作 |
| --- | --- | --- |
| 开始学习 | `setMode: study` | 切换学习模式 |
| 我要休息 | `setMode: rest` | 切换休息模式 |
| 我要出门 | `setMode: away` | 切换离家模式 |
| 开启节能 | `setMode: energy` | 切换节能模式 |
| 现在光线怎么样 | `queryLight` | 回读光照与灯状态 |
| 房间热不热 | `queryComfort` | 回读温湿度与风扇状态 |
| 安静一点 | `muteReminder` | 暂停噪声蜂鸣短提醒 |

语音失败时，网页必须保留文本测试按钮，现场可以用文本模拟同一条 MQTT `voiceIntent`。

## 7. 物理模型设计

推荐使用“学习房 + 客厅中控 + 玄关”的简化模型，而不是完整两室两厅。

```text
┌────────────────────────────┐
│ 学习房                      │
│ 光敏 + 学习灯 + 噪声传感器   │
│ 窗帘舵机 + 风扇              │
├──────────────┬─────────────┤
│ 客厅中控      │ 玄关         │
│ OLED + 8键AD  │ PIR + 蜂鸣器 │
│ N16R8 服务区  │ RGB状态灯     │
└──────────────┴─────────────┘
```

安装原则：

- 房间内只露出探头、灯、窗帘、风扇和 OLED，主控与线束放服务区或底板背面。
- 蓝色线束表示感知，橙色线束表示执行，紫色线束表示通信/显示。
- N16R8 的 USB-C 朝外，方便现场插拔和 Web Serial 授权。
- 网页看板放在模型旁边，评委能同时看到实体动作和网页变化。

## 9. 实施顺序

1. 文档冻结：确认本文件中的小学规则映射、GPIO 合同和 MQTT topic。
2. 固件最小版：实现 `hello/telemetry/ack`、光照/DHT/PIR/声音采集、学习/节能模式、灯/风扇控制。
3. Web Serial 看板：复用初中可视化框架，改成小学 profile 与 `smartlife/primary/n16r8` topic。
4. MQTT/WSS：复用初中 Cloud Relay 和 gateway，topic 前缀可配置。
5. 语音入口：先做网页文本/语音快捷指令，再接小智或服务器 STT。
6. 8 键 AD：完成模式切换与阈值焦点显示。
7. 实体模型：按学习房/客厅/玄关布局装配，隐藏线束。
8. 验收：先 mock board，再实板，最后公网 HTTPS/WSS。

## 10. 验收清单

功能验收：

- N16R8 启动后输出 `hello`，字段包含 `project=smartlife-primary` 与 `profileId=smartlife-primary-study-home-v1`。
- 遥测来自真实传感器或 mock board，不用静态假数据冒充在线。
- 光照、温湿度、噪声、PIR 在 OLED 与网页上同步变化。
- 学习模式下：光照低自动开灯，温度高自动开风扇。
- 节能模式下：无人或光照足时关闭灯/风扇，并显示节能原因。
- 网页按钮、8 键 AD、语音意图三种入口都能触发同一套命令。
- MQTT topic 能看到 `hello/telemetry/event/health/ack`。
- 断开 USB 后，网页显示离线或重连中，不显示假在线。

现场验收：

- Chrome/Edge HTTPS 页面可授权 CH340/USB 串口。
- 插板电脑网页能读写 USB；另一台设备能通过 WSS/MQTT 看到同步状态。
- 语音失败时，文本测试和快捷按钮仍能演示同一条意图链。
- 继电器只控制低压灯或指示灯。
- 风扇、舵机、蜂鸣器上电默认安全，不误转、不误响。

## 11. 主要风险与处理

| 风险 | 处理 |
| --- | --- |
| 现场 Wi-Fi 不稳定 | 第一阶段不依赖板端 Wi-Fi，使用 HTTPS Web Serial + WSS/MQTT。 |
| 浏览器不支持 Web Serial | 使用 Chrome/Edge；Safari 只做观看端。 |
| 串口被其他程序占用 | 关闭 PlatformIO Monitor、Arduino 串口监视器、本地网关后再连接。 |
| 语音识别现场噪声大 | 保留文本测试与语音快捷按钮，语音命令控制在 6-7 条。 |
| 8 键 AD 电压值不稳定 | 实板标定每个键的 ADC 区间，网页显示当前识别键。 |
| MQTT 公共 broker 不可靠 | 正式演示使用自有 Mosquitto 或初中服务器独立端口。 |
| 小学作品过于复杂 | 核心只讲四条小学任务，MQTT/可视化作为“看得见的展示平台”。 |

## 12. 最终推荐结论

推荐先做 `N16R8 智慧低碳学习小屋`，采用：

```text
小学规则主线：数据采集 -> 智能控制 -> 语音交互 -> 节能响应
真实硬件主线：N16R8 ESP32-S3 -> 光/DHT/声/PIR -> 灯/风扇/窗帘/OLED
初中功能迁移：Web Serial -> WSS Relay -> MQTT -> 可视化 Dashboard
```

这样既符合小学组智慧生活的难度和表达方式，又保留了初中项目已经跑通的 MQTT 与可视化亮点。评委看到的不是“一个网页演示”，而是一个真实小屋模型里的传感器、执行器、语音指令和远程看板全部同步工作。
